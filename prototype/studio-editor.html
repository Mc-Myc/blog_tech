Le rollback fantôme

Hier soir, migrate publishing 0004 passe. Ce matin, impossible de revenir en arrière : le rollback prétend réussir mais la colonne est toujours là.

Ce que la session a montré

Claude a commencé par lire la migration générée :

● migration 0004 ⧉
migrations.RunSQL(
    "ALTER TABLE article ADD COLUMN scene jsonb",
    reverse_sql=migrations.RunSQL.noop,  # ← le voilà
)

Le noop : le rollback « réussit » en ne faisant rien.

à retenir — un rollback qui ne peut pas échouer n'est pas un rollback

La solution

Écrire le reverse_sql explicite, et ajouter un test de migration aller-retour dans la CI…
