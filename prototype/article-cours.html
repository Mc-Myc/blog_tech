Le ND array, dessiné à la main : mes notes NumPy au propre

Au cœur de NumPy il y a un objet qui porte toutes les données du machine learning : le tableau à N dimensions. Voici mes notes de cours, schémas de cahier compris — remis au propre.

ndarray = le conteneur de TOUTES tes données ML

Un ND array, c'est une liste sous stéroïdes : indexée pareil, mais compilée en C, compacte en mémoire, et armée pour le calcul scientifique.

1D — un vecteur
2D — la matrice (ton dataset)
3D — le cube (une image RGB)
et ça continue au-delà de ton imagination — mes notes, redessinées

En pratique, 90 % du temps c'est du 2D : les lignes = les exemples de ton dataset, les colonnes = les variables (features + target). Une image en niveaux de gris ? Un 2D array de pixels 0–255. La même en couleurs ? Trois couches R, V, B empilées — ton premier 3D array.

shape est un tuple !

shape est
un tuple !
↓ immuable

L'attribut que tu liras cent fois par jour. Il donne la forme du tableau — et c'est un tuple, ce que pas mal de développeurs oublient :

2 lignes ↓
3 colonnes ⟶
ndim = 2 · shape = (2, 3) · size = 6
In deux conséquences pratiques ⧉ copier
A.shape       # (2, 3) — un tuple : indexable…
A.shape[0]    # 2 → le nombre de lignes
A.shape[1]    # 3 → le nombre de colonnes
A.shape[0] = 5  # …et IMMUABLE : TypeError.

À retenir aussi : size (le nombre total d'éléments = le produit des dimensions). Et c'est à peu près tout — ndim, tu peux l'oublier : en data-science tu vis en 2D.

reshape, squeeze, ravel — la plomberie des shapes

In le trio de manipulation ⧉ copier
K = K.reshape((3, 4))   # (2,6) → (3,4) : OK, 12 éléments des deux côtés

M = np.array([1, 2, 3])            # shape (3,)  ← le vide qui fait planter
M = M.reshape((M.shape[0], 1))    # shape (3, 1) ← le petit 1 qui sauve

O = M.squeeze()   # (3, 1) → (3,)  : fait sauter les 1
P = K.ravel()     # (3, 4) → (12,) : aplatit tout

Le (3,) avec son vide après la virgule, c'est le détail qui casse les calculs matriciels dix cellules plus loin. reshape((shape[0], 1)) pour l'ajouter, squeeze() pour l'enlever (matplotlib préfère sans), ravel() pour tout aplatir.

dtype : le détail que beaucoup ignorent

np.float16np.float64
précisionπ ≈ 3.14π ≈ 3.141592653589793
mémoire16 bits / nombre64 bits / nombre
vitesserapide — entraînements accélérésplus lent

Tous les constructeurs acceptent dtype=. Par défaut NumPy choisit bien, mais quand tu veux un code performant, c'est un levier réel : moins de bits = moins précis mais des modèles entraînés plus vite.

L'exercice — la matrice avec sa colonne de biais

exercice · tout ce qu'on vient de voir en 3 lignes Écrire initialisation(m, n) qui retourne une matrice aléatoire (m, n+1) avec une colonne de biais (remplie de 1) tout à droite. Indices : randn, ones, concatenate… et un shape.
voir la solution
● solution ⧉ copier
def initialisation(m, n):
    X = np.random.randn(m, n)
    X = np.concatenate((X, np.ones((X.shape[0], 1))), axis=1)
    return X

initialisation(3, 4)  # → shape (3, 5), dernière colonne = 1
