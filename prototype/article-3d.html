class Config:
    _instance = None

    def __new__(cls):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
        return cls._instance

Le contre-argument

Claude a pointé trois problèmes : les tests qui partagent un état global, l'import qui cache la dépendance, et le couplage invisible entre modules. Sa proposition : une injection de dépendances explicite — la config se crée une fois au démarrage, puis descend dans les composants.

Verdict

J'ai gardé l'injection. Le singleton reste au musée — dans sa vitrine 3D ci-dessus, enroulé pour toujours. Il avait raison, cette fois.
