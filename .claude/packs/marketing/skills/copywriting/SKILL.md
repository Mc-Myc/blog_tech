# Skill : copywriting

## Structures de référence

- **PAS** (Problème → Agitation → Solution) : pages de vente, emails froids. Le lecteur doit se reconnaître dans le problème avant d'entendre la solution.
- **AIDA** (Attention → Intérêt → Désir → Action) : formats longs, publicité.
- **4U** pour les titres : Utile, Urgent, Unique, Ultra-spécifique — en viser 2-3 par titre.

## Règles de fond

- **Bénéfices avant caractéristiques** : « retrouvez n'importe quel document en 3 secondes » bat « indexation full-text ». La caractéristique vient en preuve, après.
- **Un message par pièce, un CTA par page.** Deux appels à l'action se cannibalisent.
- **La preuve porte la promesse** : chiffre, témoignage, démo — une affirmation sans preuve est du bruit publicitaire.
- Écrire à la deuxième personne, langue du client (reprendre SES mots, issus des interviews/avis), phrases courtes.

## Biais cognitifs — usage légitime uniquement

| Biais | Usage légitime | Dark pattern (INTERDIT ici) |
|---|---|---|
| Preuve sociale | témoignages réels, chiffres d'usage vrais | faux avis, compteurs inventés |
| Rareté/urgence | vraie limite (stock réel, date réelle) | comptes à rebours fictifs qui se relancent |
| Ancrage | montrer la valeur comparée honnêtement | prix barré jamais pratiqué |
| Aversion à la perte | expliciter le coût réel de l'inaction | culpabilisation (« confirm-shaming ») |
| Cohérence/engagement | parcours progressif assumé | opt-out caché, cases pré-cochées |

Le critère : le lecteur, pleinement informé du procédé, se sentirait-il aidé ou
manipulé ? Manipulé = hors périmètre, quelle que soit la conversion promise.

## Sortie type

2-3 variantes par pièce avec le levier de chacune explicité (structure + biais
mobilisé) — testables en A/B, jamais « la » version unique.
